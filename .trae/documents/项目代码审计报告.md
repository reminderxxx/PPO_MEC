## 一、关键问题（高优先级）

### 🔴 1. 主车辆追踪对 Lust 数据源特殊处理
**文件**: `src/envs/core/vec_workflow_core_env.py:457-458`

```python
if self._mobility_source == "lust":
    return  # 保留旧的主车辆ID
```

**问题**: Lust 数据源下主车辆丢失时不更新主车辆ID，可能导致状态不一致。

**影响**: 在使用 Lust 数据集时，主车辆追踪可能出现异常。

**建议**: 统一处理逻辑，或添加明确的注释说明特殊处理的必要性。

---

### 🔴 2. 观测空间形状硬编码
**文件**: `src/envs/wrappers/gym_vec_env.py:72-77`

```python
self.observation_space = spaces.Box(
    low=-1.0,
    high=2.0,
    shape=(9,),  # 硬编码
    dtype=float,
)
```

**问题**: 观测空间维度固定为9，如果核心环境状态变化，这里不会自动适应。

**影响**: 扩展性问题，容易在修改环境时引入 bug。

**建议**: 从环境动态获取观测维度，或添加一致性检查。

---

### 🔴 3. Predictor Kind 逻辑反直觉
**文件**: `src/envs/core/predictor_manager.py:307-308`

```python
if self._requested_predictor_kind == "oracle":
    return "baseline"  # 请求 oracle 但返回 baseline？
```

**问题**: 当请求 predictor_kind 为 "oracle" 时，effective_predictor_kind 返回 "baseline"，逻辑看起来是反直觉的。

**影响**: 可能导致预测模式选择错误。

**建议**: 检查此逻辑是否正确，或添加详细注释说明设计意图。

---

### 🔴 4. 奖励系数硬编码
**文件**: `src/envs/core/vec_workflow_core_env.py:196-202`

```python
service_reward = 0.15
delay_penalty = 0.15
cache_miss_penalty = 1.2
migration_cost = 0.18 if not is_cross_rsu else 1.0
```

**问题**: 大量奖励系数硬编码在代码中，缺乏配置化。

**影响**: 难以进行超参数调优，实验可复现性受限。

**建议**: 将奖励系数提取到配置文件中。

---

### 🔴 5. Seed 参数被忽略
**文件**: `src/envs/wrappers/gym_vec_env.py:100-101`

```python
def reset(self, *, seed: int | None = None, options: dict[str, Any] | None = None) -> ...:
    del seed  # 被忽略
    del options  # 被忽略
```

**问题**: seed 和 options 参数被直接删除，没有使用。

**影响**: 影响实验的可复现性。

**建议**: 正确使用 seed 参数设置随机状态。

## 二、中优先级问题

### 🟡 6. 分层策略网络参数过多
**文件**: `src/agents/sa_ghmappo_core.py:254-349`

**问题**: `分层PPO基类.__init__()` 有超过 80 个参数，难以维护和理解。

**建议**: 使用配置类或字典封装相关参数。

---

### 🟡 7. LRU 淘汰默认值为极大负数
**文件**: `src/envs/core/vec_workflow_core_env.py:550-561`

```python
default=-10**9  # 如果所有 adapter 都从未被使用
```

**问题**: 默认值为 `-10**9`，如果所有 adapter 都从未被使用，会选择 ID 最小的，而非最老的。

**建议**: 使用更合理的默认值，如当前时间戳。

---

### 🟡 8. ActionAdapter 对无效动作处理不一致
**文件**: `src/envs/specs/action_schema.py:154-266`

**问题**: 不同情况下返回的无效动作结构不一致，缺乏统一策略。

**建议**: 统一无效动作处理，定义明确的契约。

---

### 🟡 9. 预测扰动应用顺序问题
**文件**: `src/envs/core/predictor_manager.py:536-593`

**问题**: 先应用噪声，再应用延迟，可能导致延迟的预测实际上是带噪声的历史预测。

**建议**: 重新审视扰动应用顺序，或添加配置选项。

---

### 🟡 10. 观测编码使用 vehicles[0] 而非 primary_vehicle_id
**文件**: `src/envs/wrappers/gym_vec_env.py:159-198`

**问题**: 使用 `vehicles[0]` 获取主车辆信息，而不是使用 `primary_vehicle_id`。

**影响**: 多车辆场景下观测可能不一致。

**建议**: 使用 `primary_vehicle_id` 获取对应车辆信息。

---

### 🟡 11. 工作流节点服务进度追踪潜在问题
**文件**: `src/envs/core/vec_workflow_core_env.py:1284-1300`

**问题**: `_advance_current_node_service()` 中 `remaining_before` 获取逻辑有潜在问题，如果 `_node_service_steps` 中不存在该节点，会返回默认值1。

**建议**: 添加断言检查节点是否存在。

---

### 🟡 12. Stall 判断条件过于严格
**文件**: `src/envs/core/vec_workflow_core_env.py:300`

```python
stall_occurred = not (primary_vehicle and base_model_ok and cache_hit and offload_target_rsu_id)
```

**问题**: 需要同时满足四个条件才不算 stall，任何一个失败都算 stall，可能导致过多的 stall 惩罚。

**建议**: 评估 stall 定义是否合理，或添加配置选项。

---

### 🟡 13. ControlAction 默认值可能导致空动作
**文件**: `src/envs/specs/semantic_objects.py:142-152`

```python
@dataclass
class ControlAction:
    cache_action: dict[str, Any] = field(default_factory=dict)
    offload_action: dict[str, Any] = field(default_factory=dict)
    migration_action: dict[str, Any] = field(default_factory=dict)
```

**问题**: 所有字段都有 `default_factory=dict`，可能导致意外的空动作。

**建议**: 考虑使用更严格的默认值或工厂函数。

---

### 🟡 14. 预测历史记录没有大小限制
**文件**: `src/envs/core/predictor_manager.py`

**问题**: `_prediction_history` 列表用于实现延迟，但没有大小限制。

**影响**: 长时间运行可能导致内存泄漏。

**建议**: 添加历史记录大小限制或定期清理。

---

### 🟡 15. 缓存目标修正逻辑可能覆盖 Agent 意图
**文件**: `src/envs/core/vec_workflow_core_env.py:608-614`

**问题**: 缓存目标修正可能导致 agent 的意图被覆盖，但只在结果中标记，没有警告。

**建议**: 添加警告日志，或让 agent 知晓修正发生。

---

### 🟡 16. 除零保护使用魔法数字
**文件**: `src/envs/core/vec_workflow_core_env.py:1281`

```python
max(self._lust_rsu_compute_scale, 1e-6)
```

**问题**: 使用 `1e-6` 作为最小值，是魔法数字。

**建议**: 定义为常量，并考虑溢出风险。

## 四、架构审计

### 4.1 模块依赖关系

```
src/agents/ → 依赖 → src/encoders/, src/trainers/
src/envs/   → 依赖 → src/data/
src/trainers/ → 依赖 → src/agents/, src/envs/
src/evaluators/ → 依赖 → src/agents/, src/envs/
```

**评估**: 依赖关系总体合理，但存在以下问题：
- `src/agents/sa_ghmappo_core.py` 文件过大（>1000行），职责过多
- `src/envs/core/vec_workflow_core_env.py` 同样过大

### 4.2 接口契约一致性

**Action Contract**: `semantic_discrete_5` 在以下文件中一致使用：
- `src/envs/specs/action_schema.py`
- `src/agents/reactive_greedy_agent.py`
- `src/agents/base_agent.py`

**Observation Contract**: 存在多个版本：
- `flat_semantic_encoder_v1`
- `flat_semantic_multi_controller_ctde_v1`
- `semantic_state_info_v1`

**建议**: 统一观测契约版本。

### 4.3 抽象层次

**问题**: 
- `分层PPO基类` 同时承担了策略网络、训练逻辑和基类接口的职责
- 建议拆分为：策略网络、训练器、Agent 包装器
